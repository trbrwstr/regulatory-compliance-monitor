# Qualified pilot protocol

## Purpose

Compare parallel manual monitoring with product output for a bounded source register. This measures workflow value and completeness; it does not establish legal or regulatory compliance.

## Required reviewer

A qualified compliance reviewer owns the pilot, confirms the source register, records the manual-monitoring method, and reviews every correction and missed item.

## Procedure

1. Start a pilot run with a named source register and reviewer.
2. Record each source item found manually and whether the product found it.
3. Record manual and product detection latency, summary corrections, and time saved.
4. Preserve source URLs and notes for each observation.
5. Close the run only after the reviewer has reconciled missed items.
6. Report manual count, product count, missed items, correction burden, latency, and time saved.

## Interpretation

Seeded fixtures and pilot observations must remain separate. A missed item blocks a completeness claim until explained. Results should justify continuing or narrowing the vertical, not support an unsupported compliance guarantee.
